## Conventions
- **Run `node smoke-test.js` after any UI/shell change.** 28 checks: pages/scripts parse, no @scope/iframe/token-cycle regressions, redirects, deep-link view init, 0 console errors, 0 overflow (desktop+mobile), a11y focus/announce, and the data-table styling (via mocked rows). Zero-dep static+HTTP checks always run; browser checks auto-find Playwright+Chromium (npx cache or global) and skip — never fail — if absent. Exit !=0 on failure. Proven to catch the deep-link regression (negative-control tested).
- Keep every tool dependency-free and local-first.
- ALL trade2 API calls go through the shared persistent adaptive queue. Never call trade2 directly.
- **Trade2 testing discipline (don't over-limit yourself):** the user WANTS live verification, not skipped tests out of caution. Before any live Trade2 call, GET `/api/trade-status` and check `limited`/`secondsRemaining`/the `rateLimit` rule states FIRST; if clear, make the spaced live calls you need; if `limited`, wait the cooldown or use `previewOnly`. Check the limit, don't avoid the API. (See memory `dont-trade-limit-during-testing`.)
- UI: premium dark dashboard. Shared design system in **theme.css** (linked first on every page) is the single source of color/type/spacing/radius/shadow/motion tokens — set tokens there, not per-page. v2 direction: NEUTRAL charcoal surfaces (near-black, ~0 chroma — never brown), gold strictly a sparse accent, real radii + soft layered shadows. Type: **Inter for all UI/headings/nav/buttons; Cinzel ONLY for the brand mark**; JetBrains Mono for data. Reusable components: `.btn`(+`-primary`/`-ghost`/`-sm`), `.badge`(+`-live`), `.card`/`.panel`, `.field`. theme.css also carries a scoped (`.appbar`/`.bar`) modern header + responsive nav that overrides per-page styles. CRITICAL: never alias a primary token to itself (e.g. `--gold:var(--gold)`) — it creates a CSS cycle that blanks the token. Pages keep only page-specific layout in their inline `<style>`; do NOT re-declare `:root` tokens.

- 2026-06-13: CORRECTED the Map Juicer mod-value model to be %-AWARE (price-vs-% curve per stat) after realising the earlier flat per-stat weight was biased. The bias: comparing stats at the same ABSOLUTE % (pack≥35 vs rarity≥35) is unfair because roll CEILINGS differ — Pack Size maxes ~41%, Item Rarity rolls to ~84%.
