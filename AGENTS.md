## Conventions
- **RESTART the local server after editing `.env` or when values look stale.** `server.js` reads `.env` (incl. `POESESSID`) **ONCE at startup** (the zero-dep loader at the top), so a changed `POESESSID`/`EE2_PROXY_BASE`/etc. does NOTHING until you stop and re-run `node server.js`. The server also spawns ONE persistent headless-PoB subprocess (via `pob.js`) that loads PoB's data once and stays resident ŌĆö if PoB itself updated (a patch) or the build values seem stale, restart the server to respawn it. **Split-host (VM uses the PC's PoB):** restart `pob-agent.js` too (`pm2 restart pob-agent`) ŌĆö it keeps PoB resident on the PC. Kill stray `node server.js` first (see the zombie-server gotcha) so you don't keep talking to an old instance with old env. (If a fresh `POESESSID` "isn't working", 99% of the time the server wasn't restarted.)
- **ALWAYS push code changes immediately after making them ŌĆö never leave them only local.** The VM deploys from `origin` (`git pull && docker compose up -d --build`), so an unpushed change is undeployable and the next session/the VM sees stale code. Edit ŌåÆ smoke-test ŌåÆ `git commit && git push` in the same session, every time. (This is the standing rule, not just a session-end nicety.)
- **Run `node smoke-test.js` after any UI/shell change.** 28 checks: pages/scripts parse, no @scope/iframe/token-cycle regressions, redirects, deep-link view init, 0 console errors, 0 overflow (desktop+mobile), a11y focus/announce, and the data-table styling (via mocked rows). Zero-dep static+HTTP checks always run; browser checks auto-find Playwright+Chromium (npx cache or global) and skip ŌĆö never fail ŌĆö if absent. Exit !=0 on failure. Proven to catch the deep-link regression (negative-control tested).
- **Browser verification must stay SILENT (no focus-stealing window) ŌĆö but it's allowed.** The user wants full capability; they just don't want a window popping up and stealing focus from their game. Full `chrome.exe` ŌĆö even headless ŌĆö flashes a focus-stealing window on Windows; the **`chrome-headless-shell.exe`** binary is genuinely windowless. `smoke-test.js`'s `chromiumExe()` now PREFERS the headless-shell, so `node smoke-test.js` runs the browser checks silently ŌĆö no window, no focus steal (verified 2026-06-24). So: prefer `curl`/API probes + reading code for most checks (faster), and use the browser when a visual/interaction check genuinely needs it ŌĆö just make sure any Playwright launch uses the headless-shell `executablePath` (mirror `chromiumExe()`), never full `chrome.exe`. `SMOKE_NO_BROWSER=1 node smoke-test.js` still exists to skip browser checks entirely if ever needed, but it's no longer required.
- Keep every tool dependency-free and local-first.
- ALL trade2 API calls go through the shared persistent adaptive queue. Never call trade2 directly.
- **Trade2 testing discipline (don't over-limit yourself):** the user WANTS live verification, not skipped tests out of caution.
